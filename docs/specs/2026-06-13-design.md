# FormalizationFlow — MVP Design

**Date:** 2026-06-13
**Status:** Alpha (breaking changes expected until stabilisation)

## Overview

FormalizationFlow is a standalone plasTeX plugin and LaTeX style that lets a mathematician write one `.tex` file and build it three ways:

- `pdflatex main.tex` → PDF
- `plastex main.tex` → HTML
- `plastex --renderer=lean4 main.tex` → sorry-filled `.lean` files

The core idea: the Lean 4 output is just another plasTeX renderer. No type checking, no type inference — each macro and environment has a fixed Lean 4 template, and the renderer walks the parse tree and emits text.

**MVP acceptance criterion:** the `unit_hom` motivating example (see below) compiles correctly in all three modes.

## Repository

Standalone GitHub repo (not inside the mono-repo). The `brainstorm/FormalizationFlow/ideas.md` file moves there. Community tool, alpha quality.

```
formalization_flow/
  Packages/
    formalizationflow.py   # plasTeX macro/environment definitions
  Renderers/
    lean4/                 # Lean 4 renderer (emits .lean files)
  sty/
    formalizationflow.sty  # LaTeX style file (for PDF via pdflatex/xelatex)
pyproject.toml
```

## Macros — object vocabulary

Each macro declares one mathematical object. Initial vocabulary:

| Macro | PDF/HTML rendering | Lean 4 output |
|---|---|---|
| `\ring{R}` | Let *R* be a ring. | `(R : Type*) [Ring R]` |
| `\commring{R}` | Let *R* be a commutative ring. | `(R : Type*) [CommRing R]` |
| `\field{K}` | Let *K* be a field. | `(K : Type*) [Field K]` |
| `\ringhom{f}{R}{S}` | Let *f* : *R* → *S* be a ring homomorphism. | `(f : R →+* S)` |
| `\unit{r}{R}` | Let *r* ∈ *R*× be a unit. | `(hr : IsUnit r)` |
| `\ideal{I}{R}` | Let *I* ⊆ *R* be an ideal. | `(I : Ideal R)` |
| `\module{M}{R}` | Let *M* be an *R*-module. | `(M : Type*) [Module R M]` |

## Environments

### `assumptions`

Collects object declarations and renders them as an itemised "Let..." list in PDF/HTML:

```latex
\begin{assumptions}
  \item \ring{R}
  \item \ring{S}
  \item \ringhom{f}{R}{S}
  \item \unit{r}{R}
\end{assumptions}
```

Renders in PDF/HTML as:
- Let *R* be a ring.
- Let *S* be a ring.
- Let *f* : *R* → *S* be a ring homomorphism.
- Let *r* ∈ *R*× be a unit.

Free-form `\item` text (not a structured macro) renders as-is in PDF/HTML and as `-- TODO: formalise assumption: <text>` in Lean 4.

### `conclusion`

The theorem's conclusion. Structured macros map to the Lean 4 return type. Free-form text maps to `True /- sketch: <text> -/` in Lean 4.

## Lean 4 renderer

Emits one `.lean` file per input `.tex` file. For each `theorem`/`lemma` environment:

```lean
theorem <label> <assumptions> : <conclusion> := by
  sorry
```

**`\label` is required.** The renderer errors clearly if a theorem or lemma has no label.

### Motivating example

Input:
```latex
\begin{theorem}\label{thm:unit_hom}
  \begin{assumptions}
    \item \ring{R}
    \item \ring{S}
    \item \ringhom{f}{R}{S}
    \item \unit{r}{R}
  \end{assumptions}
  \begin{conclusion}
    \unit{f(r)}{S}
  \end{conclusion}
\end{theorem}
```

Lean 4 output:
```lean
theorem unit_hom (R : Type*) [Ring R] (S : Type*) [Ring S] (f : R →+* S) (r : R) (hr : IsUnit r) :
    IsUnit (f r) := by
  sorry
```

## Out of scope for MVP

- **Proof structure environments** (`induction`, `cases`, `wlog`, etc.) — v2
- **Progress tracker** (sorry dashboard) — v2
- **Configurable rendering** (user-supplied plasTeX templates per environment) — v2; the itemised list rendering is the v1 default
- **Prose paragraph rendering** of assumptions — v2
- **NamespacedLatex integration** (stable cross-document tags) — deferred, depends on NamespacedLatex existing

## Related projects

- [Lean Blueprint](https://github.com/PatrickMassot/leanblueprint) — closest prior art; tracks formalization progress. No hard dependency, but intended to be combinable.
- [plasTeX](https://github.com/plastex/plastex/) — the Python LaTeX parser this builds on.
- [autospec](https://ariel-frischer.github.io/autospec/) — AI-driven test generation; potentially useful for bootstrapping macro→Lean translation tests.
